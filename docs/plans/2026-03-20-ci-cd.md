# Plan: CI/CD with GitHub Actions

**Date:** 2026-03-20
**Status:** Planned
**Goal:** Automated testing on push/PR and automated releases on version tags.

---

## Problem

The project has 204 tests that run in <1s, but nothing enforces that they pass before merging. Releases are manual. As the project grows (autonomous loop, init command, more contributors), regressions can slip in silently.

## Solution

Two GitHub Actions workflows: one for testing, one for releasing.

---

## Workflow 1: `test.yml`

### Trigger

- Push to `main`
- Any pull request targeting `main`

### Job: `test`

```yaml
runs-on: ubuntu-latest
steps:
  - Checkout repo
  - Install uv (astral-sh/setup-uv action, pinned version)
  - uv sync
  - uv run pytest tests/ -v
```

### Details

- **Python version:** 3.12 (matches `requires-python = ">=3.12"`)
- **Single platform:** ubuntu-latest. All 204 tests mock the UE connection — no platform-specific behavior to test.
- **No matrix:** One Python version, one OS. Add matrix later only if needed.
- **Timeout:** 5 minutes (tests run in <1s, but allow for uv install time)

---

## Workflow 2: `release.yml`

### Trigger

- Tag push matching `v*` (e.g., `v0.2.0`)

### Job: `release`

```yaml
runs-on: ubuntu-latest
steps:
  - Checkout repo
  - Install uv (astral-sh/setup-uv action, pinned version)
  - uv sync
  - Verify version consistency       # Check pyproject.toml, __init__.py, plugin.json match
  - uv run pytest tests/ -v          # Gate: tests must pass
  - uv build                         # Build sdist + wheel
  - uv publish                       # Publish to PyPI
  - Create GitHub Release from tag
```

### Details

- **PyPI token:** Stored as repository secret `PYPI_TOKEN`
- **Tests run first:** Release is gated on all tests passing
- **GitHub Release:** Uses `gh release create` or `softprops/action-gh-release` action with auto-generated release notes
- **No changelog automation:** Changelog is maintained manually in `CHANGELOG.md` before tagging

### Release Process (human steps)

1. Update version in `pyproject.toml` (source of truth), `ue_eyes/__init__.py`, `.claude-plugin/plugin.json`
2. Update `CHANGELOG.md`
3. Commit: `release: v0.2.0`
4. Tag: `git tag v0.2.0`
5. Push: `git push && git push --tags`
6. CI handles the rest (test → build → publish → GitHub Release)

## Files to Create

- **`.github/workflows/test.yml`** — Test workflow
- **`.github/workflows/release.yml`** — Release workflow (includes version consistency check)
- **`scripts/check_version.py`** — Script that verifies `pyproject.toml`, `ue_eyes/__init__.py`, and `.claude-plugin/plugin.json` declare the same version. Run in both workflows.
- **`CHANGELOG.md`** — Changelog file (initial entry for v0.1.0)

---

## What's Intentionally Excluded

| Item | Reason |
|------|--------|
| Windows/macOS CI matrix | Tests mock UE, no platform-specific code paths |
| Python 3.13 matrix | Single target version is sufficient for now |
| C++ plugin builds | Requires UE engine installation, not practical in CI |
| Skill validation | Skills are markdown files — low breakage risk |
| Auto-version bumping | Overhead not justified at current project scale |
| Docker builds | No containerized deployment target |
